# Auth Web Spec

Implement a login page that authenticates users and redirects back with a code (token).

---

## Flow

```
Browser opens:  <auth_url>?state=<random>&callback=http://127.0.0.1:<port>/callback
User logs in:   (your auth: form, OAuth, SSO, etc.)
Redirect to:    <callback>?state=<state>&code=<client_token>
```

---

## What to implement

### 1. Accept `?state=<state>&callback=<callback_url>` query parameters

- `state` — random CSRF token generated by the client. Must be passed back unchanged.
- `callback` — URL-encoded localhost callback URL (e.g. `http://127.0.0.1:54321/callback`)

### 2. Authenticate the user

Use whatever auth method you need. Preserve `state` and `callback` through the flow.

### 3. Redirect to callback

After successful auth, redirect to:

```
<callback>?state=<state>&code=<client_token>
```

- `state` — same value from the original request (unchanged)
- `code` — a valid client token (must work with `/projects` endpoint on your webhook server)

---

## Minimal example

```python
from fastapi import FastAPI, Query
from fastapi.responses import RedirectResponse

app = FastAPI()

@app.get("/login")
def login(state: str = Query(), callback: str = Query()):
    token = "tok_alice_abc123"  # look up from your auth
    return RedirectResponse(f"{callback}?state={state}&code={token}")
```
